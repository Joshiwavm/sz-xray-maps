# Tutorial

This walkthrough covers a full SZ + X-ray thermodynamic map analysis. A worked Jupyter notebook for ACT-CL J0459.6−4946 (z = 1.71) is in `Notebooks/Thermo_maps.ipynb`.

## Method

Thermodynamic quantities follow from the SZ Compton-y and X-ray surface brightness via an effective line-of-sight depth l_eff:

```
P_e = (m_e c² / σ_T) · y / l_eff
n_e = sqrt(1.2 · 4π(1+z)⁴ · S_X / (Λ(T) · l_eff))
T_e = P_e / n_e
K_e = T_e / n_e^(2/3)
```

where `Λ(T)` is the X-ray cooling function (erg cm³ s⁻¹) and `S_X = rate · ECF / Ω_pix`.

## Step-by-step

### 1. Initialise the Manager

```python
import numpy as np
from astropy import units as u
from SZ_xray_maps import Manager
from SZ_xray_maps.instruments import ALMA, Chandra, XMM
from SZ_xray_maps.utils import circle_mask

mgr = Manager(
    cluster_tag="J0459",
    size_arcmin=2.5,
    center=(74.923, -49.782),   # RA, Dec [deg]
    z=1.71,
    M500=4e14 * u.Msun,
    kT=7.2 * u.keV,
    kT_std=0.5 * u.keV,
    metallicity=0.37,
)
```

Cooling function tables for this cluster are generated automatically on first run and cached for subsequent calls.

### 2. Load maps

```python
# First SZ call sets the reference WCS
mgr.add_sz("band3.fits", ALMA(92e9), label="band3")

# Pass rmf/arf + z/T_keV to compute ECF from XSPEC; omit for hardcoded fallback
mgr.add_xray("chandra.fits",
             Chandra(rmf="spec.rmf", arf="spec.arf", z=1.71, T_keV=7.2),
             label="chandra", smooth_fwhm_arcsec=4.5, expmap_path="expmap.fits",
             energy_range=(0.5, 2.0))
mgr.add_xray("xmm.fits",
             XMM(rmf="xmm.rmf", arf="xmm.arf", z=1.71, T_keV=7.2),
             label="xmm", smooth_fwhm_arcsec=6.0, energy_range=(0.4, 4.0))
```

### 3. Compton-y and background

```python
mgr.compute_compton_y("band3")

bg_mask = circle_mask(mgr.xray_maps["chandra"]["data"], cx, cy, r_pix)
mgr.compute_background("chandra", bg_mask)
mgr.compute_background("xmm", bg_mask)
```

### 4. Effective line-of-sight depth

Two methods are available:

**Beta-model map** (recommended for science maps) — computes a pixel-by-pixel l_eff from a projected spherical beta model:

```python
mgr.compute_leff_map(rc_arcsec=19.5, beta=0.82)
```

**Uniform Monte Carlo** (sanity check) — solves for the single l_eff that reproduces a spectroscopic temperature prior:

```python
mgr.compute_leff_uniform("band3", "xmm", radius_arcsec=20, N=300)
```

The uniform result also returns the shape factor `C = sqrt(r500 / l_eff)`, which must satisfy `C ≥ 1` (Cauchy–Schwarz constraint).

### 5. Cross-calibration

Cross-calibrate a second instrument against a reference by matching the median projected temperature within an aperture:

```python
mgr.calibrate_xray(ref_label="xmm", target_label="chandra", sz_label="band3")
```

### 6. Thermodynamic maps

```python
mgr.compute_thermo("band3", ["chandra", "xmm"], use_leff_map=True)
```

### 7. Noise and SNR

Poisson variance is propagated analytically through the Gaussian smoothing kernel. Relative uncertainties on P_e, n_e, T_e, K_e are propagated from SZ RMS, X-ray Poisson noise, and l_eff MC samples.

```python
mgr.compute_noise()
mgr.compute_snr("band3", ["chandra", "xmm"])
```

### 8. Output

```python
mgr.diagnostics()
fig, axs = mgr.plot_thermo_grid("band3", ["chandra", "xmm"])
mgr.save_thermo_fits("output/")
```

FITS files are saved as `{cluster_tag}_{instrument}_{quantity}.fits`.

## Supported instruments

**SZ:** ALMA (any band), Mustang-2

**X-ray:** Chandra ACIS-I (0.5–2 keV), XMM-Newton EPIC MOS thin (0.4–4 keV)

To add a new instrument, subclass `SZInstrument` or `XRayInstrument`. The ECF is computed automatically from XSPEC ARF/RMF files if they are provided; otherwise supply a hardcoded fallback value (e.g. from PIMMS) for the appropriate band, spectral model, redshift, and Galactic N_H.

## Cooling function

Per-energy-bin cooling function tables are generated by PyAtomDB using the rest-frame energy bands corresponding to the observed-frame bins at the cluster redshift. Metal abundances (Z ≥ 3) are set to the specified metallicity; H and He remain at solar. Tables are identified by `cluster_tag` and `metallicity` so multiple clusters coexist in the same directory without conflict.
